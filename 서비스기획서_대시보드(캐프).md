# YSB - 은행거래 대사(Reconciliation) 대시보드 서비스기획서

> **원본 파일**: YSB_가나다회사.pbix
> **작성일**: 2026-04-20
> **총 페이지 수**: 9페이지 (Cover + Scope + Summary 2개 + 출금분석 2개 + 입금분석 2개 + 시나리오)
> **대시보드 유형**: 은행 거래내역과 회계 분개장 대사 분석 (BRA - Bank Reconciliation Analysis)

---

## 데이터 모델 구조

### 주요 테이블
| 테이블명 | 역할 | 핵심 컬럼 |
|---------|------|----------|
| **은행거래명세** | 은행에서 다운로드한 실제 거래 내역 (원본) | 출금액, 입금액, 거래처(은행), 적요(은행), 거래시간, 요일, 1stNo |
| **분개장** | 회계 시스템(ERP)의 회계 전표 분개 내역 | 적요(분개장) (계좌별 분개 금액 포함) |
| **Mapping** | 은행계좌 ↔ 분개장계좌 매핑 마스터 | 계좌코드(은행), 계좌코드(분개장), 계좌명, 구분(원화/외화) |
| **Calculation** | 모든 대사 계산 Measure를 모아둔 가상 테이블 | (DAX Measure 모음) |
| **Date_Dim** | 날짜 차원 테이블 | Date, Month Name |
| **Date_STARTDATE** | 분석 기간 시작일 | Date_STARTDATE |
| **Date_ENDDATE** | 분석 기간 종료일 | Date_ENDDATE |
| **COMPANYNAME** | 회사명 표시용 단일 값 테이블 | COMPANYNAME |

### 데이터 흐름
```
[은행 시스템]                    [ERP 회계 시스템]
   ↓                                 ↓
은행거래명세                      분개장
(실제 입출금 내역)               (장부상 입출금 내역)
   ↓                                 ↓
   └────── Mapping (계좌 매핑) ─────┘
                ↓
           Calculation
        (대사 차이 계산)
                ↓
            대시보드
   (차이 발견 → 미기표/오기표 식별)
```

### 은행거래명세 테이블 핵심 컬럼
| 컬럼 | 설명 |
|------|------|
| **출금액** | 은행 거래의 출금 금액 |
| **입금액** | 은행 거래의 입금 금액 |
| **거래처(은행)** | 은행 거래 상대방 |
| **적요(은행)** | 은행 거래 적요/메모 |
| **거래시간** | 거래 시각 (시간 단위, 시나리오 분석용) |
| **요일** | 거래 요일 (시나리오 분석용) |
| **1stNo** | 출금액의 첫 자리 숫자 (벤포드 법칙 분석용) |

### Mapping 테이블 (대사 키)
| 컬럼 | 설명 |
|------|------|
| **계좌코드(은행)** | 은행 시스템의 계좌 식별자 |
| **계좌코드(분개장)** | 분개장(ERP)의 계좌 코드 |
| **계좌명** | 사용자 친화적 계좌 이름 |
| **구분** | 원화/외화 구분 (페이지 필터 기준) |

## 공통 필터(슬라이서) 체계

| 필터명 | 위치 | 소스 | 필터 로직 |
|-------|------|------|----------|
| **계좌 선택** | 전 분석 페이지 | Mapping.계좌코드(은행) | 분석할 은행 계좌 선택 → 해당 계좌의 거래만 표시. NULL 계좌는 제외 |

| **분석 기간** | Cover/Scope | Date_STARTDATE / Date_ENDDATE | 분석 시작일/종료일 표시 (별도 단일 날짜 테이블에서 조회) |

## 주요 Measure (Calculation 테이블) 정의

> 모든 Measure는 **Calculation** 가상 테이블에 정의됨. 원화/외화 분리 구조.

### 장부 측 (분개장) 집계 Measure
| Measure명 | 산식 | 설명 |
|-----------|------|------|
| **장부 출금액 (원화)** | SUM(분개장.출금금액 WHERE Mapping.구분='원화' AND 매핑된 계좌) | 분개장에서 추출한 원화 출금 합계 |
| **장부 출금액 (외화)** | SUM(분개장.출금금액 WHERE Mapping.구분='외화') | 분개장에서 추출한 외화 출금 합계 |
| **장부 입금액 (원화)** | SUM(분개장.입금금액 WHERE Mapping.구분='원화') | 분개장에서 추출한 원화 입금 합계 |
| **장부 입금액 (외화)** | SUM(분개장.입금금액 WHERE Mapping.구분='외화') | 분개장에서 추출한 외화 입금 합계 |
| **장부 출금건 (원화)** | COUNT(분개장 출금 라인 WHERE 구분='원화') | 분개장 원화 출금 건수 |
| **장부 출금건 (외화)** | COUNT(분개장 출금 라인 WHERE 구분='외화') | 분개장 외화 출금 건수 |
| **장부 입금건 (원화)** | COUNT(분개장 입금 라인 WHERE 구분='원화') | 분개장 원화 입금 건수 |
| **장부 입금건 (외화)** | COUNT(분개장 입금 라인 WHERE 구분='외화') | 분개장 외화 입금 건수 |

### 은행 측 집계 (직접 SUM/COUNT 사용)
| 표현 | 의미 |
|------|------|
| `SUM(은행거래명세.출금액)` | 은행 거래내역의 출금 합계 (페이지 필터로 원화/외화 자동 구분) |
| `SUM(은행거래명세.입금액)` | 은행 거래내역의 입금 합계 |
| **은행 출금건** | 은행 거래내역의 출금 건수 (Calculation Measure) |
| **은행 입금건** | 은행 거래내역의 입금 건수 (Calculation Measure) |

### 대사 차이(Reconciliation) Measure — 핵심
| Measure명 | 산식 | 설명 |
|-----------|------|------|
| **출금차이(원화)** | SUM(은행거래명세.출금액) - 장부 출금액 (원화) | 은행 출금 vs 장부 출금 차이 (양수: 은행이 더 많이 출금됨, 음수: 장부에만 기록) |
| **출금차이(외화)** | SUM(은행거래명세.출금액) - 장부 출금액 (외화) | 외화 출금 차이 |
| **입금차이(원화)** | SUM(은행거래명세.입금액) - 장부 입금액 (원화) | 은행 입금 vs 장부 입금 차이 |
| **입금차이(외화)** | SUM(은행거래명세.입금액) - 장부 입금액 (외화) | 외화 입금 차이 |
| **순차이(원화)** | 입금차이(원화) - 출금차이(원화) | 순 대사 차이 (원화) |
| **순차이(외화)** | 입금차이(외화) - 출금차이(외화) | 순 대사 차이 (외화) |

### 차이 해석 가이드
| 차이값 | 의미 |
|--------|------|
| **차이 = 0** | 은행 ↔ 장부 일치 (정상) |
| **차이 > 0** | 은행에는 있으나 장부에 누락 → **분개장 미기표 가능성** |
| **차이 < 0** | 장부에는 있으나 은행에 누락 → **오기표 또는 미실행 거래 가능성** |

## 필터 연동 체계 요약

```
[페이지 필터: Mapping.구분]
    ├── '원화' → Summary(원화), 출금분석(원화), 입금분석(원화)
    └── '외화' → Summary(외화), 출금분석(외화), 입금분석(외화)

[계좌 선택 슬라이서: Mapping.계좌코드(은행)]
    ├── 전 분석 페이지 적용
    ├── NULL 제외 (매핑 안 된 계좌는 표시 안 함)
    └── 선택 시 → 해당 계좌의 은행거래명세 + 분개장 동시 필터

[Mapping 테이블 (대사 키)]
    ├── 은행거래명세 ↔ 계좌코드(은행)으로 조인
    ├── 분개장 ↔ 계좌코드(분개장)으로 조인
    └── Calculation Measure가 두 테이블을 동일 계좌 기준으로 비교

[Drill-down 흐름]
    Summary (전체 차이 인지)
        ↓
    출금/입금 분석 (월/계좌/일자별 세부)
        ↓
    은행 상세 + 분개장 상세 (원본 거래/분개 동시 비교)
        ↓
    시나리오 (이상 패턴 자동 탐지)
```

## 핵심 비즈니스 로직 — 대사(Reconciliation) 알고리즘

### 1. 데이터 정합성 (Mapping 기반)
```
은행거래명세.계좌코드(은행)  ──┐
                              ├──> Mapping (계좌 매칭)
분개장.계좌코드(분개장)      ──┘
```

### 2. 차이 계산 로직
```
For each (계좌, 기간):
    은행 출금 합계 = SUM(은행거래명세.출금액)
    장부 출금 합계 = SUM(분개장의 해당 계좌 출금 분개)
    출금차이 = 은행 출금 합계 - 장부 출금 합계

    if 차이 = 0:
        대사 일치 (정상)
    elif 차이 > 0:
        은행에 있으나 장부에 없음 (분개 누락 의심)
    elif 차이 < 0:
        장부에 있으나 은행에 없음 (오기표/미실행 의심)
```

### 3. 원화/외화 분리 처리
- Mapping.구분 컬럼으로 사전에 계좌를 원화/외화로 분류
- 페이지 필터로 자동 분리 → 외화 환산 없이 통화별 직접 비교
- 외화 계좌의 경우 SUM(은행거래명세.출금액)이 외화 단위 그대로 집계됨

## Insights — 범용화 검토

### 양호한 점
- **Mapping 테이블 기반 설계**: 계좌 코드 매핑이 데이터로 분리되어 있어, 새 회사 적용 시 Mapping 테이블만 갱신하면 자동 동작
- **원화/외화 분리**: `Mapping.구분` 컬럼으로 통화 구분을 데이터화 → 통화 추가 시 페이지 복제만으로 확장 가능
- **Calculation 가상 테이블**: 모든 Measure를 한 곳에 집중 → 산식 변경 용이
- **하드코딩 없음**: 특정 계좌명/거래처명을 코드에 직접 참조하는 부분 없음

### 개선 제안
| 영역 | 현재 | 개선 방안 |
|------|-----|----------|
| **시나리오 임계값** | "이중지불 = COUNT ≥ 2" 등 임계값이 차트 필터에 하드코딩 | **시나리오설정 테이블**로 분리 (예: 이중지불_최소건수, 영업시간_시작/종료, 벤포드_허용오차) |
| **벤포드 법칙** | 출금액 첫자리 분포 표시만 | 자연 분포 대비 **편차율 자동 계산 Measure** 추가하여 임계값 초과 시 경고 표시 |
| **영업시간 정의** | "거래시간"별 표시만 (사용자가 직관적으로 판단) | `is_영업시간` 계산 컬럼 추가 (회사별 영업시간 설정 테이블 참조) |
| **통화 확장** | 원화/외화 2종 (페이지 복제 방식) | 통화별 동적 페이지 생성 또는 통화 슬라이서 도입 |
| **차이 임계값** | 모든 차이 표시 | **유의차이 임계값** 설정 (예: 1만원 미만 차이는 무시) → 사용자 설정 가능 |

### 신규 회사 적용 절차
```
1. Mapping 테이블 생성/갱신
   ├── 은행 시스템의 계좌코드 목록
   ├── ERP의 계좌코드 목록
   └── 매칭 + 원화/외화 구분

2. 데이터 소스 연결
   ├── 은행거래명세 (CSV/Excel/API)
   └── 분개장 (ERP 추출)

3. Date 차원 갱신 (분석 기간 설정)
4. 회사명 변경 (COMPANYNAME 테이블)
5. 시나리오 임계값 검토 (회사별 영업 패턴 반영)
```
